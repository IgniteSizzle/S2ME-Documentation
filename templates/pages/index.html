    Добро пожаловать

    В этом разделе мы сделаем первые шаги в мире Java ME (Micro Edition) — узнаем, что это за платформа, зачем она нужна, как выглядит базовый синтаксис Java и в чём особенности разработки для устройств с ограниченными ресурсами.

    Наша цель — понять суть Java ME и получить общее представление о том, как пишутся и работают простейшие приложения.

    Что такое Java ME

    Java ME (Micro Edition) — это облегчённая версия платформы Java, предназначенная для запуска программ на устройствах с невысокой производительностью и ограниченным объёмом памяти.
    Ранее Java ME широко использовалась для создания приложений под кнопочные телефоны (например, игры и утилиты формата .jar), её можно было встретить в:

    • Мобильных телефонах
    • Смарт-карты и RFID-устройства
    • Планшеты и КПК
    • Мобильные операционные системы
    • Мобильные игровые приставки и мультимедийные устройства
    • Терминалы для бизнеса и точек продаж

    Приложения Java ME называются MIDlet (от Mobile Information Device applet). Они выполняются внутри специальной виртуальной машины — KVM (Kilo Virtual Machine), которая поддерживает лишь часть стандартных возможностей Java.

    Основы синтаксиса Java

    Java ME основана на языке Java, поэтому её синтаксис почти полностью совпадает с обычной Java SE. Однако в Java ME доступны не все стандартные библиотеки, а возможности интерфейса, сети и файловой системы ограничены.
    Тем не менее, базовые принципы те же:

    1. Классы и объекты

    Java — это объектно-ориентированный язык. Всё состоит из классов и объектов.
    Каждое приложение Java ME содержит главный класс MIDlet, который наследуется от javax.microedition.midlet.MIDlet. 

    
		public class HelloWorldMIDlet extends MIDlet {
			// код приложения
		}

    2. Точки входа в программу

    В отличие от обычной Java-программы (где используется public static void main()), MIDlet управляется системой.
    В нём есть три ключевых метода жизненного цикла: 

    
		public void startApp()    // вызывается при запуске
		public void pauseApp()    // вызывается при сворачивании/приостановке
		public void destroyApp(boolean unconditional) // вызывается при завершении

    3. Работа с интерфейсом

    Для вывода информации на экран Java ME использует пакеты javax.microedition.lcdui.*.

    Пример простейшего интерфейса: 

    
		import javax.microedition.lcdui.*;
		import javax.microedition.midlet.*;

		public class HelloWorldMIDlet extends MIDlet {
			private Display display;

			public HelloWorldMIDlet() {
				display = Display.getDisplay(this);
			}

			public void startApp() {
				Form form = new Form("Привет, Java ME!");
				form.append("Добро пожаловать в мир мобильной Java!");
				display.setCurrent(form);
			}

			public void pauseApp() {}
			public void destroyApp(boolean unconditional) {}
		}

    4. Переменные и типы данных

    Java ME поддерживает все базовые типы данных Java:

    # Тип Описание Пример
    1 byte 8-битное целое число со знаком byte b = 127;
    2 short 16-битное целое число со знаком short s = 32767;
    3 int 32-битное целое число со знаком int i = 2147483647;
    4 long 64-битное целое число со знаком long l = 9223372036854775807L;
    5 float 32-битное число с плавающей точкой float f = 3.14f;
    6 double 64-битное число с плавающей точкой double d = 3.14159;
    7 char 16-битный символ Unicode char c = 'A';
    8 boolean логический тип, принимает значения true или false boolean flag = true;
    9 String последовательность символов (объект) String str = "Привет";

    5. Управляющие конструкции

    Java ME полностью поддерживает стандартные конструкции Java: 

    
		if (x > 0) {
			System.out.println("Положительное число");
		} else {
			System.out.println("Отрицательное число");
		}

		for (int i = 0; i < 5; i++) {
			System.out.println("i = " + i);
		}

    6. Методы и классы

    Методы объявляются внутри классов. Например: 

    
		public class MathUtils {
			public int add(int a, int b) {
				return a + b;
			}
		}

    Преимущества и недостатки JavaME

    Не стоит забывать что JavaME имеет положительные и отрицательные характеры.
    Ниже мы расмотрим все положительные и отрицательные стороны разработки на JavaME!


    Преимущества Java ME:

    • Кроссплатформенность — программы могут работать на разных устройствах без изменений.
    • Безопасность — изолированная среда выполнения (песочница).
    • Небольшой размер — приложения занимают мало памяти.
    • Строгий синтаксис и стабильность, как у полной Java.

    Недостатки Java ME:

    • Ограниченный набор API и библиотек.
    • Отсутствие современных инструментов и фреймворков.
    • Трудности с совместимостью разных устройств и версий профилей.
